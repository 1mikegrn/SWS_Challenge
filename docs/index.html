SWS_Challenge: A submission for the Super Word Search Challenge¶

This site is a docstring repository for the Super Word Search Challenge, written by Michael Green in part for a technical challenge submission to Karagozian and Case.

Use the tabs on the left-hand side of the page to navigate to the various document sections.

Getting Started¶

From the command prompt, the latest version this library can be installed via pip and git.

pip install git+https://github.com/1mikegrn/SWS_Challenge

Where the setup file will automatically check dependencies and install to the main module library. Once installed, calling SWS_Challenge <input.txt> or SWS_Challenge <grid.txt> <words.txt> <condt> from the command prompt will execute the calculation.

The <input.txt> file should be the full path to the input file as specified by the challenge instructions. This file should be of format:

N M
XXX
XXX
XXX
P
ZZZ
ZZZ
ZZZ

where N M is N-row M-column dimensionality of the grid of letters in space XXX, and P is the length of the words in space ZZZ.

Alternatively, the grid and words can be provided in separate .txt files, with <grid.txt> for the XXX grid and <words.txt> for the ZZZ words respectively, with the <condt> tag for wrapping appended to the end of the console command as WRAP or NO_WRAP

Library Structure¶

SWS_Challenge/
    __init__.py                         # initialize SWS_Challenge
    __main__.py                         # CLI access point
    check_space.py                      # checks space b/t ctr and bounds
    super_word_search.py                # handles CLI to module interaction
    src/                                # source module
        __init__.py                     # initialize src
        grid.py                         # grid class
        targets.py                      # target words class
        input_file.py                   # parse CLI input
        tools/                          # tools module
            __init__py                  # initialize tools
            cmd_reader.py               # CLI reader
            build_test_grid.py          # script to build test grids
            interpolate_space.py        # interpolate grid space

NNA_Challenge¶

SWS_Challenge.__main__.main()¶

main CLI entry point, parses CLI inputs according to necessary module parameters

SWS_Challenge.check_space.check_space(grid, ctr, letters)¶

checks the space in the grid for the set of letters which extend from the first letter’s center space, through the length of the word.

SWS_Challenge.super_word_search.find_word(word, grid_class)¶

handle word finding in check_space, parse response

SWS_Challenge.super_word_search.main(grid_class, word_class)¶

main function, brings everything together

class SWS_Challenge.src.targets.Targets(data_input)¶

target object for the words passed to module

SWS_Challenge.src.input_file.parse_input(file_name)¶

parse input in accordance to the challenge parameters. Accepts a single input file and partitions data to a len(3) tuple for the grid, words, and wrap argument.

SWS_Challenge.src.tools.cmd_reader.reader()¶

reader which parses CLI arguments

SWS_Challenge.src.tools.build_test_grid.build_test_grid(n_rows, n_cols, words_to_hide, output_path, wrap)¶

build dummy test files

SWS_Challenge.src.tools.interpolate_space.interpolate_space(ctr, bound, word_length, n_rows, n_cols, wrap)¶

interpolates space from center node to bounding node. returns the list of index values which are between the center node and the bound node.
